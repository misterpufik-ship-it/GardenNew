Инструкция для сверки расходов БН

Общее правило

1. Сверять каждое юрлицо отдельно: файлы Лаунджа брать из папки "Лаундж", файлы Вымпела - из папки "Вымпел".
2. В отчетах брать только расходы, где вид оплаты равен "бн" и сумма заполнена.
3. Дату расхода брать из текущего дневного блока отчета: дата указана в колонке A в начале блока.
4. При сверке можно пренебречь копейками: суммы сравнивать по рублям, без учета копеек. Например, 1 250,00 и 1 250,75 считать совпадением, если совпадает рублевая часть.
5. Сначала сопоставлять расходы отчета с выпиской по сумме в рублях, с учетом повторяющихся сумм.
6. Если есть несколько вариантов совпадения, сначала выбирать операцию с той же датой, затем любую еще не использованную операцию с той же суммой в рублях.
7. После совпадений один-в-один проверять групповые совпадения в рамках одного дня и одного юрлица:
   - брать одну неиспользованную строку выписки;
   - смотреть неиспользованные расходы отчета за этот же день и это же юрлицо;
   - если отдельный расход не совпал со строкой выписки, попробовать сложить несколько расходов отчета за этот день;
   - если сумма этих расходов по рублям совпадает с суммой строки выписки, считать это групповым совпадением.
8. При групповых совпадениях копейками также можно пренебречь: сравнивать сумму группы расходов и сумму выписки по рублям, без учета копеек.
9. Каждому групповому совпадению присваивать отдельную цифру/номер группы: 1, 2, 3 и далее.
10. Один и тот же номер группы ставить:
   - у всех строк расходов отчета, которые сложены в группу;
   - у строки выписки, с которой они сопоставлены.
11. Все строки группового совпадения окрашивать в желтый цвет:
   - желтым окрашивать все расходы отчета, вошедшие в сумму группы;
   - желтым окрашивать строку выписки, с которой эта группа совпала;
   - рядом с расходами и строкой выписки указывать одинаковую цифру группы.
12. Цвета сверки:
   - зеленый - точное совпадение один-в-один;
   - желтый - групповое совпадение нескольких расходов отчета с одной строкой выписки, отмеченное одинаковой цифрой группы;
   - оранжевый - совпадение не найдено.
13. Итог выдавать отдельным файлом с дашбордом, деталями сверки, расходами отчета и выпиской.

Лаундж

1. В файле "отчет Никита.xlsx" брать расходы из основного листа:
   - категория: колонка F;
   - сумма: колонка G;
   - вид оплаты: колонка H;
   - комментарий: колонка I.
2. В файле "выписка апрель 2026.xlsx" брать банковские операции с дебетом:
   - дата: строка блока "Дата: ...";
   - номер операции: колонка B;
   - контрагент: колонка C;
   - назначение платежа: колонка F;
   - дебет: колонка J.
3. Сопоставлять колонку "Сумма" отчета с колонкой "Дебет" выписки.

Вымпел

1. Для Вымпела использовать файлы из подпапки "Вымпел".
2. В файле "отчет Никита11.xlsx" обязательно учитывать оба листа отчета:
   - "У_Апрель_(1 смена)";
   - "А_Апрель (1 смена)".
3. На обоих листах брать строки расходов по одной структуре:
   - категория расходов: колонка F;
   - сумма: колонка G;
   - вид оплаты: колонка H;
   - комментарий: колонка I.
4. В итоговом дашборде обязательно сохранять колонку "Лист", чтобы было видно, с какого листа отчета взята каждая строка.
5. В документе "СберБизнес_Выписка_за_2026_04_01_2026_04_30.docx" брать операции по дебету:
   - дата проводки;
   - счет;
   - контрагент;
   - сумма по дебету;
   - номер документа;
   - назначение платежа.
6. Сопоставлять колонку "Сумма" отчета с колонкой "Сумма по дебету" выписки по рублям, без учета копеек.
7. Групповые совпадения Вымпела искать только внутри одной даты и только по операциям Вымпела. Если несколько расходов отчета за один день в сумме по рублям совпадают с одной строкой выписки, окрашивать эти расходы и строку выписки в желтый цвет и ставить им одинаковый номер группы.
8. Дашборд Вымпела выдавать отдельным файлом в подпапке "Вымпел" с листами: дашборд, расходы отчета, дебет выписки, детальная сверка и методика.
