# Безопасная публикация сайта

## Как связаны локальный проект, GitHub, SSH и хостинг

Локальная папка - это место, где мы редактируем сайт.

Git - это история изменений в этой папке. Каждый commit является точкой, к которой можно вернуться.

GitHub - это удаленная копия Git-истории. Он нужен, чтобы история не жила только на одном компьютере, и чтобы можно было видеть, какая версия была отправлена.

Хостинг/сервер - это удаленный компьютер, где лежит рабочий сайт. Обычно его публичная папка называется `public_html`.

SSH - это защищенный способ подключиться к серверу. Сам SSH не хранит версии и не делает откаты; он просто дает безопасный канал, по которому можно выполнить команды и отправить файлы.

## Рекомендуемый порядок

1. Изменить сайт локально.
2. Проверить изменения.
3. Сделать commit в Git.
4. Отправить commit на GitHub.
5. Запустить деплой на хостинг.

Скрипт `scripts/deploy.ps1` делает шаги 4-5 и перед обновлением сайта создает резервную копию текущего `public_html` на сервере.

## Разовая настройка

1. Скопировать пример конфига:

```powershell
Copy-Item scripts/deploy.env.example.ps1 scripts/deploy.env.ps1
```

2. Заполнить в `scripts/deploy.env.ps1`:

```powershell
$DeployHost = "host-or-ip"
$DeployUser = "ssh_user"
$DeployPort = 22
$RemotePath = "/absolute/server/path/to/public_html"
```

Этот файл добавлен в `.gitignore`, поэтому SSH-данные не попадут в GitHub.

3. Проверить подключение:

```powershell
ssh -p 22 ssh_user@host-or-ip
```

4. Сделать пробный прогон без публикации:

```powershell
.\scripts\deploy.ps1 -DryRun
```

5. Опубликовать:

```powershell
.\scripts\deploy.ps1
```

## Откат

Показать доступные резервные копии:

```powershell
.\scripts\rollback.ps1 -List
```

Откатиться на последнюю копию:

```powershell
.\scripts\rollback.ps1
```

Откатиться на конкретную копию:

```powershell
.\scripts\rollback.ps1 -Backup "/home/username/site/_deploy_backups/public_html-20260609-120000.tgz"
```

## Важные правила безопасности

- Не деплоить незакоммиченные изменения, кроме аварийных случаев.
- Не хранить пароли, приватные ключи и CMS-конфиги в GitHub.
- Перед первым деплоем убедиться, что `$RemotePath` указывает именно на папку сайта, а не на домашнюю папку пользователя.
- Если на сервере есть загружаемые пользователями файлы, база данных или CMS-кеш, их нужно держать вне деплой-пакета или отдельно добавить в исключения.
- Откат файлов не откатывает базу данных. Для базы нужны отдельные backup/export правила.
